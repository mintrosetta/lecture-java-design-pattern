Method Factory
    เมื่อใดก็ตามที่เราต้องการย้าย logic การสร้างวัตถุของเรา

    เมื่อใดก็ตามที่ใช้ method factory เราไม่จำเป็นต้องรู้ว่าจะสร้าง instance คลาสอะไรไว้ล่วงหน้า
    และเราสามารถเพิ่มคลาสใหม่มาได้เรื่อย ๆ ต่อไปได้ โดยที่เราไม่ต้องกลับมาแก่ไขโค้ด
    
    โดยเราจะมอบหมายการสร้างวัตถุให้กับคลาสย่อย โดยคลาสย่อยจะตัดสินใจว่าจะสร้างวัตถุใด

    ข้อเสีย
        - เพิ่มความซับซ้อน เพราะจำเป็นต้องใช้คลาสจำนวนมากขึั้น และคลาสเหล่านี้จำเป็นต้องได้รับการทดสอบ
        - เราต้องออกแบบ method factory ไว้ตั้งแต่ต้น เพราะไม่ใช่เรื่องง่ายที่จะปรับโครงสร้าง


    สรุป
        method factory design pattern ใช้แบบนี้เมื่อเราต้องการมอบหมายการสร้าง instance ให้กับคลาสลูก
        และต้องการจัดการกับการสร้างวัตถุใหม่โดยไม่ต้องแก้ไขโค้ดชุดเดิม (simple factory ต้องแก้ไข if else)

        เปลี่ยจาก 
            การตัดสินใจว่าจะสร้าง instance อะไร โดยการรับ parameters เข้ามาเช็คเงื่อนไข
            เป็นขึ้นอยู่กับ sub class ว่า sub class ได้สร้าง instance ของคลาสอะไรให้

            ตัวอย่าง
                (simple factory)
                    Post post = PostFactory.create("news"); // ได้รับ instance ของ PostNews
                (method factory)
                    PostCreator postCreator = new PostNewsCreator();
                    Post post = postCreator.getPost(); // ได้รับ instance ของ PostNews
